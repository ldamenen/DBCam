      What this PoC does not do

      This is a throwaway demo to validate the core loop, not the shippable product. It deliberately does not provide:

      • Encryption at rest — no Secure-Enclave / Keystore key storage.
      • Always-on / background recording — browsers suspend hidden tabs; the screen wake lock is best-effort and can be released by the OS.
      • Encrypted, tamper-proof sealing — this PoC does demonstrate the flow (raw recorded continuously, incidents sealed as segments, authorized unseal + audit log), but the raw blob is not encrypted at rest and the "authorization" is a UI gate, not split-key cryptography. That hardening is native-only.
      • Jurisdiction enforcement — the Policy Engine is an injectable stub and does not resolve or enforce real law.

      These guarantees are native-only (iOS Vision/CryptoKit, Android ML Kit/Keystore). The animal deterrent may help avoid, definitely helps document — it is not a safety guarantee.
